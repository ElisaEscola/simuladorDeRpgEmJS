                                          📝 O DESAFIO
Vocês foram contratados para desenvolver a lógica de combate de um novo jogo de RPG indie. O objetivo é criar um script em JavaScript que simule uma luta entre um Herói e um Monstro no console do navegador. A batalha deve acontecer automaticamente até que um dos dois fique sem vida!

>>>PASSO A PASSO PARA O DESENVOLVIMENTO

Fase 1: Criando os Personagens (Variáveis)

O primeiro passo é definir quem são os combatentes. Vocês devem criar variáveis para armazenar o "estado" de cada um.
***Dica: Criem variáveis para o Nome, a Vida e o Poder de Ataque do Herói e do Monstro.
-->Conceito avaliado: Uso correto de let e tipos primitivos (String e Number).

Fase 2: A Máquina de Ataque (Funções)

Não queremos repetir código toda vez que alguém bater! Criem uma função chamada realizarAtaque.
A função deve receber quem está atacando e quanto de dano ele dá.
Ela deve subtrair o dano da vida do alvo e mostrar no console.log o que aconteceu.
-->Conceito avaliado: Criação de funções, parâmetros e retorno.

Fase 3: O Campo de Batalha (Loops)

A luta não pode ter apenas um turno. Vocês devem usar uma estrutura de repetição para que o Herói e o Monstro troquem ataques até que a vida de alguém chegue a zero.
***Dica: Usem o loop while (Enquanto vida > 0).
-->Conceito avaliado: Lógica de repetição e condições de parada.

Fase 4: O Grande Vencedor (Condicionais)

Ao final do loop, o sistema deve verificar quem sobrou e exibir uma mensagem épica de Vitória ou Game Over.
-->Conceito avaliado: Estruturas if / else.

                        📊 CRITÉRIOS DE AVALIAÇÃO (O que o Prof vai olhar)
Funcionamento (2,0 pts): O código roda sem erros no console? A batalha tem um início, meio e fim?

Uso de Conceitos (2,0 pts): O grupo aplicou Variáveis, Funções, Loops e Condicionais de forma correta?

Defesa do Projeto (1,0 pt): Durante a aula, passarei nos grupos e farei uma pergunta técnica para os integrantes. Saber explicar o que o seu código faz é fundamental!
